# Contribution policy

Contributions are welcome when they improve this public documentation without weakening its boundary.

## Requirements

A proposed contribution must:

- remain conceptual, generic, and documentation-only;
- comply with the [public/private boundary](../docs/public-private-boundary.md);
- avoid claims that this repository contains software or an operational implementation;
- preserve accurate attribution, DOI metadata, and license notices;
- use clear Markdown and identify supporting public sources where appropriate;
- receive maintainer review before merge.

Do not submit secrets or restricted information, even for demonstration or redaction. Because Git history can retain removed content, notify maintainers through an appropriate private channel if sensitive material is exposed; do not repeat it in a public issue.

## Licensing note

The published documentation is licensed under `CC-BY-NC-ND-4.0`. That license prohibits recipients from distributing modified versions of the licensed material. A contribution proposal does not itself change the repository license, and acceptance is at the rights holder's discretion. Contributors must have the right to submit their material and should expect attribution and licensing questions to be resolved before acceptance.

## Review outcome

Maintainers may approve, request revision, or decline a proposal. No contribution is merged automatically.
