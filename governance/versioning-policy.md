# Versioning policy

Version labels identify reviewed snapshots of public documentation. They do not represent software releases, deployment readiness, security certification, or the maturity of any private implementation.

## Change classes

- **Major:** substantial changes to the public conceptual model or document organization.
- **Minor:** meaningful public additions or clarifications that preserve the model.
- **Patch:** corrections to language, links, citation metadata, or formatting.

These classes are documentation conventions rather than a promise of strict Semantic Versioning compatibility.

## Publication controls

Before identifying a new public version:

1. review all changed files against the public/private boundary;
2. verify authorship, license, and DOI metadata;
3. confirm that no software-release claim is implied;
4. record human approval for publication;
5. create or update an archival record only through a separately authorized process.

Use the version DOI for a specific archived publication and the concept DOI when referring to all versions. Repository edits do not automatically create a Zenodo version.
