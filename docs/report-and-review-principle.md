# Report-and-review principle

Every bounded task ends with a handoff. The worker reports; the supervisor reviews; the human-governed process decides what happens next.

## Worker report

A useful report states:

- access and execution status
- actions actually performed
- files or artifacts created, changed, drafted, or skipped
- evidence used and checks run
- constraints and stop conditions encountered
- uncertainties, risks, and unresolved items
- a recommendation that does not presume approval

The report must distinguish fact from inference and completed work from proposed work.

## Supervisor review

The review checks the report against the task card and asks:

- Did the worker remain within scope?
- Is the claimed output present and reviewable?
- Were required checks completed?
- Did any public/private, safety, legal, or access gate trigger?
- Are uncertainties visible and proportionate?
- Would approval create a false or misleading claim?

## Outcomes

- **Approve:** accept the bounded result for the stated purpose.
- **Revise:** issue a specific, bounded correction task.
- **Stop:** close or escalate because a gate, risk, or missing prerequisite prevents safe continuation.

Approval of one task does not authorize later publication, deployment, merge, or release unless that action is separately within scope and human-approved.
